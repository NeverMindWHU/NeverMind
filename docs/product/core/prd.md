## 4. 产品概述

### 典型使用场景

<!-- IMPORTANT -->

#### 场景1：截图、自动笔记与知识卡片

Jack选了一门概率论与数理统计课。Jack正在看网课自学，同时打开了NeverMind软件在后台运行。此时课程视频中出现了一个重要的公式：t分布的构造。Jack原本的学习方法是，暂停视频，拿出平板电脑，在GoodNotes上抄下公式，到了期末再统一复习。但是有了NeverMind，Jack只需要暂停视频，调用NeverMind的全局截图快捷键，在屏幕上框选出公式，点击回车，然后继续看网课（可能还会停下来熟悉一下公式）。运行在后台的NeverMind笔记软件会自动调用LLM进行图片识别，自动提取出公式，并生成两样东西：1) 笔记：AI自动识别当前Jack的工作区，找到相关的笔记并在其中添加笔记内容；2）知识卡片：生成一个标题为“t分布的构造”的知识卡片，在卡片内添加公式和其他内容。Jack继续一边看网课以便截取重要知识点，当Jack看完后， 把NeverMind调到前台，发现程序已经自动帮Jack把所有截取下来的知识点整理成了笔记和一张张知识卡片。Jack对生成的卡片进行了检查，发现没有问题，于是将卡片都入库了。

期末到了，没有下载NeverMind的Amy由于过于忙碌，没有安排好学习计划，发现自己把上半学期教的内容全部忘光了，于是只能从头开始复习，浪费了大量时间。

而Jack早在半个月前就因为软件的复习提醒完成了基于艾宾浩斯遗忘曲线的多轮复习，现在Jack已经基本掌握了所有相关概念。不过Jack觉得不够，于是Jack又手动调出了知识卡片，让软件帮他生成了一些自测题，进行了进一步的查漏补缺。

半年后，软件又一次基于遗忘曲线提醒Jack巩固一下数理统计的知识点。Jack轻松地把所有知识点都过了一遍，半年前的记忆得到了进一步的加深。

一年后，Jack由于科学的复习计划还记得数理统计的所有内容，在写论文时想起来一个重要的结论，把公式应用到了论文中。Amy没有复习，她只能模糊地记得公式的大概样子，却想不起来公式的分母是什么了。

#### 场景2：多输入源

Paul经常使用LLM进行专业知识的问答。他发现LLM的回答非常准确，但是由于他与LLM经常进行对话，导致他的对话历史十分混乱，积累了大量的知识点。久而久之，Jack发现，有一个知识点明明在很久之前问过LLM，但是他却怎么也想不起来了，因为他在那一次对话以后就没有再看过这个对话。他想要找这个对话，却发现聊天列表有上百个会话历史，根本找不到。

后来Jack安装了NeverMind，并在浏览器上安装了NeverMind的插件。每当他在与LLM进行对话，觉得当前的知识点有必要记下来的时候，他就会直接点击插件添加在LLM输出框上的“添加到NeverMind”按钮。NeverMind会自动读取当前页面中的对话内容，将原对话内容进行保存并提取知识点、生成笔记和知识卡片。两周后，由于遗忘曲线的作用，Jack收到了NeverMind的复习提醒。Jack重温了一次当时的对话内容，把这个知识点牢固地记住了。

### 产品定位

基于AI的自动化笔记软件与复习工具。

**产品亮点**：
- 最小化操作地进行知识点记录
- 基于科学记忆法的智能复习提醒
- 基于AI的自动复习辅助

**业务目标**：

- 打造"AI生成+科学复习+知识管理"一体化平台

**用户目标**：

- 减轻记忆负担，减少手动整理时间
- 遵循科学记忆法，提高学习效率
- 统一管理知识卡片，方便随时查看复习

### 4.3 目标用户

| 用户群体 | 使用场景               | 核心需求                     |
| -------- | ---------------------- | ---------------------------- |
| 学生     | 考试备考、课堂笔记整理 | 知识点快速生成、科学复习计划 |
| 职场人士 | 技能提升、知识沉淀     | 高效记忆、随时查阅           |

### 4.4 用户痛点

#### 1. 采集阶段：高摩擦力导致的“心流中断”
*   **手动记录效率极低**：在看视频或网课时，用户需频繁暂停（如场景中的Jack），手动切换到笔记软件输入公式或概念。这种频繁的任务切换（Task-Switching）会严重破坏学习心流。
*   **富媒体转换困难**：公式、图表、代码等内容，手动录入成本极高，导致用户往往选择“只看不记”，最终看过即忘。
*   **碎片化对话的流失**：在与LLM对话时，大量高质量的知识点散落在数百个会话历史中（如场景中的Paul）。缺乏一键沉淀的机制，导致“向AI提问”变成了“一次性消费”，知识无法进入个人知识库。

#### 2. 管理阶段：知识沉淀后的“黑洞化”
*   **知识孤岛与搜索疲劳**：笔记散落在GoodNotes、Notion、浏览器书签及LLM会话历史中。当用户需要提取某个具体结论时，往往记不清存放在哪，导致“知识黑洞”现象——记过，但找不着。
*   **笔记与记忆脱节**：传统笔记是静态的，用户往往“只管记、不管看”。积累了大量精美笔记，却从未真正进入大脑。

#### 3. 复习阶段：违背科学规律的“低效勤奋”
*   **记忆周期的不可控**：绝大多数用户（如场景中的Amy）依赖“临时抱佛脚”式的冲刺复习。这种方式违背艾宾浩斯遗忘曲线，导致在考前或应用前需要花费巨大的时间“重新预习”，而非“复习”。
*   **复习计划的管理成本**：虽然Anki等软件支持艾宾浩斯复习，但手动制作卡片、设置复习间隔的门槛太高。用户往往在“管理复习计划”上花费了比“复习本身”更多的时间，导致最后放弃。

#### 4. 应用阶段：关键时刻的“精度缺失”
*   **模糊记忆的无能为力**：在写论文、技术开发或商务谈判等关键时刻，用户往往能记得“学过这个”，但无法回忆起核心细节（如公式的分母、参数的定义）。这种记忆的“精度缺失”导致知识无法转化为生产力。
*   **缺乏自测闭环**：用户自以为“看懂了”就是“记住了”，缺乏基于知识点的即时自测手段，导致知识在脑中缺乏深度连接。

#### 总结：核心痛点映射表

| 痛点环节 | 表现形式 | NeverMind 解决方案 (亮点) |
| :--- | :--- | :--- |
| **获取** | 暂停录入、心流中断、LLM对话散落 | **全局截图+插件一键抓取**，AI自动识别生成 |
| **管理** | 笔记积压、找不到会话、整理耗时 | **AI自动归档与笔记/卡片双重生成** |
| **记忆** | 记了就忘、复习无序、临时抱佛脚 | **自动化艾宾浩斯排程**，强制对抗遗忘 |
| **应用** | 细节模糊、重学成本高、无自测 | **卡片精准召回+AI生成自测题** |

### 4.5 主要功能

<!-- IMPORTANT -->

1. **智能笔记生成与卡片整理**：多输入源自动生成结构化知识卡片
2. **AI定时复习**：基于艾宾浩斯记忆法推送复习提醒
3. **知识库**：自动分类整理所有知识卡片与笔记

## 5. 功能需求详情

### 5.1 采集模块

采集模块是用户录入知识点的入口，目标是尽可能降低"记录"这个动作的操作成本，让用户在不中断学习心流的前提下完成知识沉淀。

#### 5.1.1 全局截图采集

用户可在全局任意界面通过快捷键唤起截图覆盖层，框选屏幕上任一区域（公式、图表、代码段落等），截图确认后，AI自动识别图片内容并提取知识点。

这是整个产品的核心功能，优先级最高。

**用户操作流程**：
1. 快捷键唤起截图覆盖层
2. 框选目标区域
3. 确认截取，回到学习原场景（如继续看网课）
4. 后台AI自动处理截图，生成笔记和知识卡片
5. 用户回到应用预览生成结果，确认入库

**关键设计要点**：
- 截图覆盖层需快速响应（毫秒级），不可影响用户使用其他应用
- 支持一次截取中包含多个知识点（AI自动拆分）
- 支持连续多次截图，批量处理

#### 5.1.2 浏览器插件采集

用户在与LLM对话或浏览网页时，通过浏览器扩展一键将当前内容发送到NeverMind。

插件在LLM聊天界面的输出框旁注入"添加到NeverMind"按钮，点击后自动捕获对话内容，AI读取上下文提取知识点，生成卡片。

**用户操作流程**：
1. 在LLM对话中看到有价值的知识点
2. 点击浏览器插件按钮
3. 内容自动发送到NeverMind
4. AI提取知识点并生成卡片，进入预览待确认状态

#### 5.1.3 手动输入

用户在应用内手动输入文本或关键词，AI据此生成知识卡片。适用于没有富媒体来源、纯概念性知识点的场景。

支持图文混合输入——用户可以输入描述的同时附上参考图片。

#### 读取剪贴板

<!-- IMPORTANT -->

用户选中文本复制后，调用一个全局快捷键，AI自动读取剪贴板内容并生成卡片。适合快速记录零散的文本信息。

无论通过何种方式采集，所有输入最终都由AI处理并生成结构化的知识卡片。

AI根据输入内容自动生成一张或多张知识卡片，每张卡片包含：
- **核心关键词**：标识知识点的名称
- **问题描述**：以疑问句形式表述该知识点（如"什么是t分布？"）
- **定义**：简明扼要的概念定义
- **详细解释**：展开说明，帮助理解
- **相关术语**：与此知识点关联的其他概念
- **应用场景**：该知识点在什么情况下会被用到
- **来源摘录**（可选）：原始输入片段，帮助追溯来源
- **助记词**（可选）：便于记忆的联想提示

AI同时会根据知识点内容自动打标签，为后续检索和归类做准备。

#### 5.2.2 预览与确认

生成的卡片不会直接进入知识库，而是先进入预览待确认状态。

**预览功能**：
- 以批次为单位展示生成的卡片（一次截图/输入可能产生多张卡片）
- 用户可逐张查看卡片内容
- 支持单张接受或拒绝
- 支持批量接受或拒绝
- 接受后可在知识库模块中编辑补充

这一设计的目的是让用户拥有最终决定权，确保入库内容质量，同时降低AI幻觉带来的风险。

### 5.3 复习模块

复习模块是产品的核心价值所在。AI根据艾宾浩斯遗忘曲线自动安排复习计划，用户无需手动管理复习节奏。

#### 5.3.1 自动化复习排期

每张被接受的卡片自动纳入艾宾浩斯复习排程。系统按照预定的遗忘曲线间隔（1天→1天→3天→7天→15天→30天）为每张卡片安排复习时间。

**排期规则**：
- 新接受的卡片从第1轮开始，1天后需要复习
- 复习时根据用户的反馈调整排期：
  - **记住**：按正常节奏进入下一轮，间隔递增
  - **遗忘**：退回第1轮重新开始
  - **跳过**：保持当前轮次，推迟到合适的复习时间
  - **完成**：不再纳入排期
- 系统自动处理所有排期计算，用户无需关心

#### 5.3.2 复习流程

**用户操作流程**：
1. 打开复习页面，看到今日待复习数量和完成进度概览
2. 逐张进行复习
3. 卡片正面展示问题（疑问句），用户回忆答案
4. 翻转卡片，查看答案和详细解释
5. 用户自我判断并选择：记住了 / 忘了 / 跳过 / 完成
6. 提交后自动进入下一张，或结束本轮复习

**翻卡交互设计**：
- 正面只显示问题，背面显示完整答案——模拟传统闪卡体验，强制用户先回忆再确认
- 翻转动画清晰传达"展示答案"的动作
- 四个结果按钮醒目排列，降低操作成本

#### 5.3.3 复习概览

复习首页展示关键指标：
- 今日待复习数量
- 今日已完成数量
- 连续学习天数
- 下一个即将到期的卡片信息

#### 5.3.4 提前复习

用户可随时查看尚未到期但即将到期的卡片总数，并选择提前开始复习，不等系统安排。

#### 5.3.5 复习通知

系统通过桌面通知提醒用户进行复习：
- 通知时间可自定义（默认每天上午9:00）
- 通知内容包含今日待复习数量
- 用户可独立控制总通知开关和复习提醒开关

### 5.4 知识库模块

知识库是所有已入库卡片的集中管理中心，提供搜索、筛选、浏览、编辑和管理能力。

#### 5.4.1 浏览模式

知识库提供两种浏览视角：

**关键词桶视图**：按主关键词将卡片分组展示，每个词桶显示包含的问题数量和最后更新时间。适合宏观浏览某个主题下的所有知识点。

**搜索视图**：通过关键词或自然语言问题搜索卡片，支持全文检索。适合快速定位特定知识点。

#### 5.4.2 筛选与排序

- 按标签过滤
- 按状态筛选（全部/活跃/已归档）
- 按创建时间、更新时间、下次复习时间、关键词排序

#### 5.4.3 卡片管理

**单张编辑**：用户可对已有卡片进行修改，包括修改关键词、定义、解释、标签、相关术语、应用场景和助记词等。

**归档与恢复**：用户可将不再关注的卡片归档，归档后卡片不再出现在复习排期中，但仍保留在知识库中可随时查阅。

**批量操作**：支持批量归档、恢复、添加标签、移除标签。

#### 5.4.4 导入导出

支持将卡片导出为JSON或CSV格式，方便备份和迁移。同时支持从文件导入卡片数据。

### 5.5 知识图谱模块

知识图谱通过AI自动分析知识点之间的语义关联，构建可视化的知识网络，帮助用户发现知识点之间的联系，形成结构化的知识体系。

#### 5.5.1 自动关联

每当我们生成新的知识卡片时，系统自动进行两项关联工作：
- AI在生成卡片时自动提取相关术语，建立语义关联
- 系统通过嵌入向量计算新卡片与已有卡片的相似度，自动发现关联关系

#### 5.5.2 图谱可视化

以力导向图展示知识点之间的关联网络：
- 节点代表知识点，大小反映被关联的次数（越核心的知识点越大）
- 节点颜色按标签或主题分类
- 连线代表知识点之间的关联，强度越高连线越粗
- 支持拖拽调整节点位置
- 点击节点可查看关联的卡片详情

#### 5.5.3 关联检索

在卡片详情页中，展示该卡片的所有关联卡片列表，帮助用户顺藤摸瓜式地扩展知识网络。

### 5.6 设置模块

#### 5.6.1 通用设置

用户可配置：
- 应用主题（浅色/深色/跟随系统）
- 通知开关和复习提醒开关
- 复习提醒时间
- 默认AI模型

#### 5.6.2 模型配置

用户可添加和管理多个AI模型配置，支持主流大模型API（OpenAI Compatible、通义千问等）：
- 添加、编辑、删除模型配置
- 设置默认模型
- 测试模型连通性（验证API是否可达及延迟情况）
- API Key安全存储

#### 5.6.3 存储设置

配置导出文件的默认保存路径。

#### 5.6.4 危险操作

提供数据库清理功能，允许用户一键清除所有本地数据：
- 需二次确认（输入确认文字）
- 明确提示操作不可撤销

## 6. 非功能需求

### 6.1 性能

- 应用冷启动时间控制在3秒以内
- AI卡片生成响应时间不超过30秒（单批次）
- 卡片列表浏览流畅，加载无明显卡顿
- 截图覆盖层唤起延迟极低，不影响用户体验

### 6.2 安全

- API Key通过系统安全存储（如macOS Keychain）保管，不以明文持久化
- 所有用户数据存储在本地，不上传至任何第三方服务器（AI调用除外）
- 截图数据仅在当次AI请求中使用，不作持久化存储

### 6.3 可用性

- 核心功能（知识库浏览、复习）支持离线使用
- 数据持久化采用本地SQLite数据库
- AI请求失败时不丢失用户输入数据
- 支持 macOS、Windows、Linux 三大平台

### 6.4 数据容量

- 本地支持存储万级数量的知识卡片，不影响浏览和检索体验

## 7. 技术架构概述

### 7.1 技术栈

- **前端**：React + TypeScript + Vite + Tailwind CSS
- **桌面框架**：Tauri (Rust)
- **数据库**：SQLite（本地）
- **AI**：支持 OpenAI Compatible 协议的多模型接入（通义千问等）
- **浏览器扩展**：Chrome/Firefox 扩展

### 7.2 模块划分

| 模块     | 路由       | 说明                     |
| -------- | ---------- | ------------------------ |
| 首页     | `/`        | 总览、快捷入口             |
| 卡片生成 | `/generate`| 输入、预览、接受/拒绝      |
| 截图     | `/screenshot`| 截图覆盖层界面           |
| 复习     | `/review`  | 复习流程、概览             |
| 知识库   | `/library` | 卡片搜索、筛选、管理       |
| 知识图谱 | `/graph`   | 知识点关联图谱可视化       |
| 设置     | `/settings`| 应用设置、模型配置         |

### 7.3 前后端分工

- 前端负责UI展示、用户交互和状态管理
- 后端（Rust/Tauri）负责数据持久化、AI调用、复习排期计算、系统通知
